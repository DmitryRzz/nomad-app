# NOMAD — Roadmap

## Этап 1: Discovery & MVP Scope
**Цель:** Определить, что реально войдёт в MVP. Отсечь всё лишнее.

- Анализ конкурентов: Google Travel, TripIt, TripAdvisor, AllTrails, GetYourGuide
- Определение core-loop: что пользователь делает первым делом, когда открывает приложение
- User stories: 3-5 ключевых сценариев (например: "приехал в город, хочу провести день эффективно")
- MVP scope lock: что входит, что откладывается

**Выходной артефакт:** PRD (Product Requirements Document) — 3-5 страниц.

---

## Этап 2: Архитектура и Техстек
**Цель:** Выбрать стек, который позволит двигаться быстро и масштабироваться.

- Frontend: React Native / Flutter (кроссплатформа)
- Backend: Node.js + Fastify / Python + FastAPI
- База данных: PostgreSQL + PostGIS для геоданных, Redis для кеша
- AI/ML: OpenAI API для планирования, Whisper для голоса
- AR: ARCore (Android) / ARKit (iOS) — или WebAR для прототипа
- Карты: Mapbox / OpenStreetMap + собственный tile-сервер
- Хостинг: AWS / GCP / Hetzner

**Выходной артефакт:** Техническая архитектура (диаграммы + описание API).

---

## Этап 3: Core Engine — AI-Планировщик
**Цель:** Сделать главную фичу — интеллектуальный маршрут на день.

- Интеграция с LLM (OpenAI / Claude) для генерации маршрутов
- Контекстные переменные: погода, время суток, энергия, бюджет, предпочтения
- Интеграция с картами: построение маршрута между точками
- Fallback: если AI недоступен — шаблонные маршруты

**Выходной артефакт:** Работающий прототип: вводишь город → получаешь маршрут на день.

---

## Этап 4: Smart Compass (AR-наложение)
**Цель:** Показать релевантную информацию через камеру телефона.

- Прототип: точки интереса с метками поверх камеры
- Геолокация + гироскоп для позиционирования меток
- Фильтрация: показывать только то, что в маршруте или предпочтениях
- Источник данных: Google Places API + собственная база

**Выходной артефакт:** Демо: наводишь камеру на улицу → видишь 3-5 меток.

---

## Этап 5: Социальный слой
**Цель:** Найти людей с похожими интересами.

- Геолокационный поиск: кто рядом, какие интересы
- Триггеры событий: "идём вместе на концерт"
- Групповой чат для маршрута
- Модерация и безопасность

**Выходной артефакт:** Матчинг работает: находит людей по интересам в радиусе.

---

## Этап 6: Языковой щит
**Цель:** Убрать языковой барьер.

- Whisper для распознавания речи
- OpenAI для перевода с контекстом
- TTS для озвучки
- Фото-меню: OCR + описание блюд

**Выходной артефакт:** Голосовой диалог: говоришь на русском — собеседник слышит на местном.

---

## Этап 7: Интеграции и Монетизация
**Цель:** Подключить реальные сервисы и заработать.

- Бронирование отелей/экскурсий (GetYourGuide API, Booking.com API)
- Платёжная система: Stripe + локальные методы
- Сплит-оплата между пользователями
- Pro-подписка: freemium-модель

**Выходной артефакт:** Можно забронировать и оплатить через приложение.

---

## Этап 8: Полировка и Запуск
**Цель:** Продукт готов к публикации.

- UI/UX ревью: путь пользователя от установки до ценности
- Тестирование: бета с 50-100 пользователями
- ASO (App Store Optimization)
- Публикация в App Store / Google Play

**Выходной артефакт:** Приложение в сторах, первые пользователи.

---

## Зависимости между этапами
```
Этап 1 → Этап 2 → Этап 3 → Этап 4
                          ↓
                     Этап 5 → Этап 6
                          ↓
                     Этап 7 → Этап 8
```

Этапы 4, 5, 6 можно вести параллельно после завершения Этапа 3.
