## Часть 2. Upstream пулы и инфраструктура

### Критические находки

#### UP-001: Воркер Online но аренда завершена
- **Описание:** T-Hash показывает воркер "1" как Online с 93.8 TH/s, но аренда MRR #5511294 завершена 14 часов назад (04/24 07:44 am).
- **Влияние:** Ложный статус вводит пользователя в заблуждение. Данные могут быть фейковыми.
- **Исправление:** Синхронизировать статус с MRR API.

#### UP-002: Расхождение хешрейта до 37%
- **Описание:**
  - T-Hash: 93.8 TH/s
  - EMCD: 100.07 TH/s
  - MRR: 108-131 TH/s
- **Исправление:** Унифицировать источник данных. Показывать все 3 метрики с пометками.

### Проблемы высокого приоритета

- **UP-003:** Продолжительность аренды — пользователь указал "40 часов", фактическая аренда MRR = 24 часа
- **UP-004:** Имя воркера не соответствует — T-Hash: "1", MRR: "xminer-1.2.6"

### Проблемы среднего приоритета

- **UP-005:** EMCD средний хешрейт за день = 9.02 TH/s при текущем ~100 TH/s (~90% даунтайм)
- **UP-006:** Нет статуса "Аренда завершена" в T-Hash

### Низкий приоритет

- **UP-007:** Нет интеграции с MRR API для статуса аренды

## Часть 3. Административная панель

### Критические проблемы

#### ADM-SEC-001: Пароли upstream пулов в открытом виде
- **Страница:** Admin → Edit Group Pool Configs
- **Описание:** Видны пароли для EMCD: **"123"** (дважды) и **"x"**. Логины тоже видны: andersen1.testth, andersen1.whatsminer, mark80.worker.
- **Влияние:** Любой админ может перенаправить хешрейт или украсть средства.
- **Исправление:** Удалить пароли из UI. Использовать HashiCorp Vault или AWS Secrets Manager.

#### ADM-SEC-002: Простые пароли для upstream connections
- **Описание:** Пароли "123" и "x" — тривиальные. Дают прямой доступ к EMCD аккаунтам.
- **Исправление:** Сгенерировать сложные пароли. Включить 2FA на EMCD. Ротация.

### Высокие проблемы

- **ADM-001:** T-Hash — полная прокси для EMCD. Все 8 конфигов ведут на gate.emcd.io / gate.emcd.network. Нет собственной инфраструктуры.
- **ADM-002:** Нет fallback при отказе пула. 3 конфига с приоритетами но нет авто-переключения.
- **ADM-SEC-003:** Имена конфигов содержат персональные данные ("emcd Dima test").

### Средние проблемы

- **ADM-UI-001:** Server time без таймзоны
- **ADM-FUN-001:** Add Payment History без валидации формата
- **ADM-DAT-001:** Таблица Users показывает email пользователей (частично маскированы)
- **ADM-UI-002:** Equipment Group "Default" имеет нереалистичный хешрейт (9 квадриллионов)

### Низкие проблемы

- **ADM-UI-003:** Dropdown "Default" единственный вариант
- **ADM-UI-004:** Кнопка "Add group" непонятная
- **ADM-UI-005:** Delete без подтверждения (можно случайно удалить production config)
