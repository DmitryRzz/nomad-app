## 1. Анализ конкурентов

### 1.1 Google Travel / Google Trips
**Что делает:** Объединяет перелёты, отели, маршруты в одном интерфейсе.
**Сильные стороны:**
- Глубокая интеграция с Gmail (автоимпорт бронирований)
- Офлайн-карты
- Бесплатно
**Слабые стороны:**
- Пассивный инструмент — не даёт рекомендаций
- Не адаптируется под контекст (погода, энергия)
- Нет социального слоя

### 1.2 TripIt
**Что делает:** Автоматически строит маршрут из бронирований.
**Сильные стороны:**
- Автоимпорт из email
- Timeline-view всей поездки
- Pro-версия с альтернативами рейсов
**Слабые стороны:**
- Только организация, не открытия
- Нет AI-рекомендаций
- Устаревший UI

### 1.3 TripAdvisor
**Что делает:** Отзывы, рейтинги, бронирование экскурсий.
**Сильные стороны:**
- Огромная база отзывов
- Viator для бронирования
- Форумы сообщества
**Слабые стороны:**
- Перегружен рекламой
- Отзывы часто фейковые/устаревшие
- Нет персонализации

### 1.4 GetYourGuide / Viator
**Что делает:** Маркетплейс экскурсий и активностей.
**Сильные стороны:**
- Широкий выбор
- Отзывы и рейтинги
- Instant-booking
**Слабые стороны:**
- Только коммерческие активности
- Нет локальных/спонтанных событий
- Не интегрированы в маршрут

### 1.5 AllTrails
**Что делает:** Маршруты для хайкинга.
**Сильные стороны:**
- 400K+ маршрутов
- Офлайн-карты
- GPS-навигация
**Слабые стороны:**
- Только природа, не города
- Нет интеграции с другими активностями

### 1.6 Rome2Rio
**Что делает:** Показывает все способы добраться из точки А в Б.
**Сильные стороны:**
- Уникальная база данных транспорта
- Мультимодальные маршруты
**Слабые стороны:**
- Только транспорт, не опыт
- Нет контекста

### 1.7 MAPS.ME / Organic Maps
**Что делает:** Офлайн-карты на OpenStreetMap.
**Сильные стороны:**
- Полностью офлайн
- OpenStreetMap — бесплатно
- Пешие/велосипедные маршруты
**Слабые стороны:**
- Только карты, не рекомендации
- Устаревшие POI

## 2. Core Loop

**Вопрос:** Что пользователь делает первым делом, открыв приложение?

**Вариант A — Планировщик:**
1. Вводит город и даты
2. Получает готовый маршрут на день
3. Редактирует / свайпает "не нравится"
4. Сохраняет и следует маршруту

**Вариант B — Discovery:**
1. Открывает карту текущего местоположения
2. Видит метки "рядом сейчас"
3. Тапает интересное
4. Добавляет в маршрут или идёт сразу

**Вариант C — Гибрид (рекомендуем):**
1. При открытии — пульс города: "что происходит рядом"
2. Если запланирована поездка — показывает маршрут
3. Если нет — предлагает открытия
4. Один тап — добавить в маршрут

## 3. User Stories

### US-001: Приехал в город
> Как турист, приехавший в незнакомый город, я хочу получить готовый маршрут на день, чтобы не тратить время на планирование.

**Критерии приёмки:**
- Ввод города → маршрут за < 5 секунд
- Маршрут включает 4-6 локаций
- Учитывает время суток (не кафе в 23:00)
- Показывает расстояние между точками

### US-002: Погода изменилась
> Как турист, я хочу, чтобы маршрут адаптировался под дождь, чтобы не стоять под зонтом у открытой достопримечательности.

**Критерии приёмки:**
- Дождь → indoor-активности
- Жара → тень/кондиционер
- Холод → крытые рынки/музеи

### US-003: Хочу компанию
> Как одиночный путешественник, я хочу найти людей со схожими интересами рядом, чтобы не исследовать город в одиночку.

**Критерии приёмки:**
- Фильтр по интересам
- Радиус поиска
- Безопасная коммуникация
- Групповые мероприятия

### US-004: Не понимаю язык
> Как турист в стране с незнакомым языком, я хочу переводить меню и разговоры в реальном времени.

**Критерии приёмки:**
- Фото меню → перевод
- Голосовой диалог
- Офлайн-режим для базовых фраз

### US-005: Хочу спонтанности
> Как опытный путешественник, я хочу находить локальные события (не в путеводителе), чтобы видеть настоящую жизнь города.

**Критерии приёмки:**
- Локальные события, не экскурсии
- Popup-рынки, уличные выступления
- Джем-сейшны, винные дегустации

## 4. MVP Scope Lock

### Входит в MVP:
- [ ] AI-планировщик маршрутов (1 день, 4-6 точек)
- [ ] Базовая карта с POI
- [ ] Адаптация под погоду (indoor/outdoor)
- [ ] Голосовой перевод базовых фраз
- [ ] Фото-меню (OCR + перевод)
- [ ] Сохранение маршрутов
- [ ] Профиль с предпочтениями

### Откладывается (после MVP):
- [ ] AR Smart Compass
- [ ] Социальный матчинг
- [ ] Групповые чаты
- [ ] Сплит-оплата
- [ ] Бронирование через приложение
- [ ] Pro-подписка
- [ ] Партнёрские комиссии

### Решение:
MVP = **AI-планировщик + адаптивность + языковой щит**

Это даёт core-ценность: приехал → получил маршрут → говоришь на местном → не тонешь в дожде.

Всё остальное — усиление.
